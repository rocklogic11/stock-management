### 3.4 数据库写入性能

**修复前** (❌ 存在问题):

| 测试项 | 总耗时 | 成功数 | 失败数 | 成功率 | 评价 |
|--------|--------|--------|--------|--------|--------|
| 单条商品创建 | 24ms | 1/1 | 0 | 100% | ✅ 正常 |
| 批量商品创建(10条) | 94ms | 1/10 | 9 | 10% | ❌ 严重 |
| 批量商品创建(50条) | 337ms | 1/50 | 49 | 2% | ❌ 严重 |

**问题根因**: `generateSkuCode()` 在并发时生成重复SKU

**修复方案**: 使用 `crypto.randomBytes()` 生成唯一SKU

**修复后** (✅ 测试通过):

| 测试项 | 总耗时 | 成功数 | 失败数 | 成功率 | 平均每条 | 评价 |
|--------|--------|--------|--------|--------|----------|--------|
| 单条商品创建 | 31ms | 1/1 | 0 | 100% | 31ms | ✅ 正常 |
| 批量商品创建(10条) | 250ms | 10/10 | 0 | 100% | 25ms | ✅ 正常 |
| 批量商品创建(50条) | 1045ms | 50/50 | 0 | 100% | 21ms | ✅ 正常 |

**根本原因分析**:
1. **SKU生成函数存在并发问题** (`generateSkuCode()`)
   - 代码位置: `server/src/utils/helpers.js` 第38-51行
   - 问题: 多个并发请求同时查询最后一个商品的SKU，导致生成相同的SKU
   - 结果: 违反唯一约束，写入失败

2. **代码示例**:
```javascript
// 问题代码 (helpers.js 第38-51行)
const generateSkuCode = async (Product) => {
  const lastProduct = await Product.findOne({
    order: [['id', 'DESC']],
    attributes: ['sku_code'],
  });
  let nextNum = 1;
  if (lastProduct && lastProduct.sku_code) {
    const match = lastProduct.sku_code.match(/KW-TOY-(\d+)/);
    if (match) {
      nextNum = parseInt(match[1]) + 1;
    }
  }
  return `KW-TOY-${String(nextNum).padStart(3, '0')}`;
};
```

3. **并发执行流程**:
```
请求1: 查询最后SKU → 得到 "KW-TOY-005" → 计算 nextNum=6
请求2: 查询最后SKU → 得到 "KW-TOY-005" → 计算 nextNum=6  ← 重复！
请求3: 查询最后SKU → 得到 "KW-TOY-005" → 计算 nextNum=6  ← 重复！
```

## 5. 优化建议 (可选)

### 5.1 优化并发读取性能

1. **添加Redis缓存**
   - 缓存商品列表、分类列表等读多写少的数据
   - 预计可提升QPS 3-5倍

2. **数据库查询优化**
   - 为`product_name`, `sku_code`, `category_id`添加索引
   - 使用`select`限制返回字段，减少数据传输

### 5.2 数据库升级建议

- 当前使用SQLite，适合小型应用
- 如果用户量增长，建议迁移到MySQL或PostgreSQL
- 获得更好的并发支持和性能
