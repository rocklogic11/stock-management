# ============================================
# 生产环境部署检查清单
# ============================================

## 部署前检查

### 1. 代码准备 ✅
- [ ] 代码已从测试环境合并到主分支
- [ ] 所有测试已通过
- [ ] 版本号已更新
- [ ] 构建无错误 (`npm run build` 成功)

### 2. 环境配置 ✅
- [ ] 创建 `.env.production` 文件
- [ ] **修改默认JWT密钥** (必须!)
- [ ] 配置数据库连接信息
- [ ] 配置Redis (如果使用)
- [ ] 配置CORS允许的域名
- [ ] 确认 `NODE_ENV=production`

### 3. 安全配置 ✅
- [ ] JWT_SECRET 已修改为随机字符串 (≥32字符)
- [ ] JWT_REFRESH_SECRET 已修改为随机字符串 (≥32字符)
- [ ] 数据库密码强度足够 (≥12字符，含特殊字符)
- [ ] 关闭了开发环境的调试功能
- [ ] 配置了HTTPS (SSL证书)
- [ ] 防火墙只开放必要端口 (80, 443, 22)

### 4. 数据库准备 ✅
- [ ] 已创建生产数据库
- [ ] 已执行建表SQL
- [ ] 已创建数据库用户并授权
- [ ] 数据库备份策略已制定
- [ ] 数据库字符集正确 (utf8mb4)

### 5. 服务器准备 ✅
- [ ] 操作系统已更新
- [ ] Node.js 已安装 (v18+)
- [ ] PM2 已安装
- [ ] Nginx 已安装并配置
- [ ] (可选) Redis 已安装并运行
- [ ] 磁盘空间充足 (≥10GB)
- [ ] 内存充足 (≥2GB)

## 快速验证命令

```bash
# 1. 检查后端健康
curl http://localhost:3000/api/health

# 2. 检查PM2状态
pm2 status

# 3. 检查Nginx状态
sudo systemctl status nginx

# 4. 检查数据库
mysql -u username -p -e "SELECT 1"

# 5. 检查端口监听
sudo lsof -i :80
sudo lsof -i :3000

# 6. 检查磁盘空间
df -h

# 7. 检查内存使用
free -h
```
