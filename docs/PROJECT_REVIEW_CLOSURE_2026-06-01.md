# 项目评审优化闭环记录（2026-06-01）

## 结论

最近两天提出的、会影响当前核心业务闭环的修改项已完成并自测通过。核心闭环包括：商品新增/编辑、手机实时扫码、拍照识别兜底、4 张商品图片上传、PC/手机双端展示、分类下拉、后端数据校验、数据库迁移、接口文档/实施日志。

## 已完成项

### 商品扫码与图片

- 新增/编辑商品支持商品编号 `barcode`。
- 手机端实时扫码已切换为 `getUserMedia + video playsinline + ZXing BrowserMultiFormatReader`。
- iOS Safari HTTPS 实时扫码已由人工测试确认成功。
- 拍照识别改为 ZXing 主识别，叠加原图、增强图、旋转图、旋转增强图，并保留多识别器兜底。
- 所有图片识别器增加超时保护，避免长期停留“正在识别码...”。
- 商品图片最多 4 张，前端和后端均有限制。
- 新增/编辑取消时清理本次未保存上传图片；保存后清理已移除的旧图片。

### 后端接口与数据一致性

- `product.barcode` 增加唯一性校验和唯一索引迁移。
- `product.images` 支持图片数组存储。
- 新增/编辑商品接口校验商品编号重复、图片数量、图片 URL 来源。
- 商品图片上传接口和删除接口已补齐。
- 入库/盘点扫码接口支持按 SKU 或商品包装条码匹配。
- 分类接口修复异步缓存缺少 `await` 的问题，避免返回 `{}` 导致前端下拉为空。
- 商品、分类、库存、仪表盘相关缓存清理链路已补齐到主流程。

### PC 与手机端体验

- 商品档案 PC 端保留表格模式。
- 商品档案手机端改为卡片列表。
- 商品新增/编辑手机端使用全屏弹窗，底部固定操作按钮。
- 商品图片上传手机端使用 2x2 宫格。
- 商品新增按钮权限恢复逻辑已修复，刷新后不会因权限状态丢失而隐藏。
- 分类下拉增加前端防御：接口异常时不再把非数组数据写入下拉，新增商品打开时会主动补拉分类。

### 测试环境

- PC 推荐使用 `http://localhost:3000/`。
- 手机普通功能推荐使用 `http://192.168.0.102:3000/`。
- iPhone 实时扫码必须使用 HTTPS；当前使用 Cloudflare Quick Tunnel 转发到 `http://127.0.0.1:3000`。
- 已避免 Cloudflare 转发到 Vite dev server 导致加载慢的问题。

## 仍建议后续专项优化

以下不阻断当前核心业务测试，但建议作为后续迭代处理：

- 前端主包仍有超过 500KB 的构建警告，建议后续做 Element Plus/vendor 分包。
- `npm audit` 曾提示依赖漏洞，建议单独安排依赖升级和回归测试。
- 首页驾驶舱已经有 KPI、分类库存结构、待办风险、健康排行，但如果要达到更完整的“经营驾驶舱”效果，建议后续增加趋势图、库存周转、补货优先级矩阵和移动端图表布局专项优化。
- 入库/盘点工作台当前仍偏 PC 工作台形态，若要求手机扫码作业同等重要，建议后续把入库/盘点也接入统一扫码组件。
