# 移动端商品录入体验优化记录 - 2026-06-03

## 范围

本轮执行 P1-4：移动端扫码、拍照、表单、弹窗体验强化。

## 已完成

1. 商品新增/编辑弹窗
   - 手机端全屏弹窗继续保留。
   - 底部操作区固定，增加 iOS safe-area 适配。
   - 保存按钮增加 loading，避免弱网下重复提交。
   - 分类加载中禁止保存，分类为空时继续保留重试入口。
   - 无商品管理权限时，不再显示编辑、上下架、二维码等操作入口。

2. 实时扫码
   - 手机端扫码弹窗改为全屏。
   - 扫码预览区高度更稳定。
   - 底部关闭按钮适配 iOS safe-area。
   - 失败提示下增加“重新扫码”和“拍照识别”快捷入口。

3. 拍照/相册上传商品图片
   - 将“拍照添加”和“相册选择”拆成两个明确入口。
   - 拍照 input 使用 `capture="environment"`。
   - 相册 input 支持多选。
   - 上传中禁用入口，避免重复提交。
   - 继续限制最多 4 张商品图片。

4. 全局移动弹窗
   - 修正全局 `.el-dialog` 移动端规则，避免影响 Element Plus 全屏弹窗。

## 待用户 UAT

请在 iPhone 上验证：

1. 打开线上系统。
2. 登录店主账号。
3. 进入商品档案。
4. 点击新增商品。
5. 验证弹窗是否全屏、底部按钮是否可点击且不被 Safari 底栏遮挡。
6. 点击实时扫码，验证摄像头权限、预览画面、扫码填充商品编号。
7. 点击拍照识别，验证拍摄条码照片后的识别结果或失败提示。
8. 点击拍照添加商品图片，验证缩略图和保存。
9. 点击相册选择，验证可选择多张图片且最多 4 张。
10. 保存商品后，回到列表确认商品出现。

## 剩余风险

1. 生产地址仍是 HTTP over IP，iOS 实时扫码长期需要 HTTPS。
2. 拍照识别受照片清晰度、反光、条码占比影响，仍应保留手动输入路径。
3. 本轮只优化商品新增/编辑，入库和盘点扫码工作台的移动端深度改造仍在后续 UX 待办中。
