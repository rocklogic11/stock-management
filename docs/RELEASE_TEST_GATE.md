# 发布前测试门禁

更新时间：2026-06-03

## 适用范围

适用于本项目从当前版本进入生产发布前的 SIT、UAT、冒烟测试、回归测试与上线后观察。

## 门禁结论规则

| 结论 | 含义 |
| --- | --- |
| Pass | 可进入下一阶段 |
| Block | 不允许发布，需要修复 |
| Need UAT | 自动化通过，但需要用户真机/真实业务确认 |

发布条件：

1. 自动化门禁必须 Pass。
2. PC 与 iOS 手机核心流程必须 UAT Pass。
3. 不得存在 P0/P1 未关闭缺陷。
4. 发布前必须有备份和回滚方案。

## 自动化门禁

本地执行：

```powershell
npm run release:check
```

覆盖范围：

| 检查 | 说明 | 阻塞发布 |
| --- | --- | --- |
| 数据库迁移 | `migrate-add-barcode-images` 幂等执行 | 是 |
| 后端语法 | 关键路由、权限工具、测试脚本 | 是 |
| 前端构建 | Vite 生产构建 | 是 |
| 发布冒烟 | 登录、分类、商品、图片、仪表盘 | 是 |
| P0 回归 | 入库两阶段、盘点快照、敏感字段过滤 | 是 |

生产服务器执行：

```bash
cd /opt/stock-management/server
npm run env:check:production -- .env.production
```

## 缺陷分级

| 等级 | 定义 | 发布决策 |
| --- | --- | --- |
| P0 | 数据错误、登录不可用、库存错误、密钥泄露 | 禁止发布 |
| P1 | 核心流程失败、手机端无法新增商品、扫码/上传完全不可用 | 禁止发布 |
| P2 | 非核心体验问题，有明确替代路径 | 可评估发布 |
| P3 | 文案、样式、小体验问题 | 可发布后修复 |
