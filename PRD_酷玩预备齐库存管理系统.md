# 库存预备齐 - 库存管理系统产品需求文档(PRD)

> 文档版本：v1.0 MVP  
> 创建日期：2026-04-28  
> 产品名称：库存预备齐  
> 产品版本：V1.0（MVP版本）  
> 撰写人：产品经理（AI辅助）

---

## 一、产品概述

### 1.1 产品定位
面向玩具店的库存管理系统，通过扫码技术实现快速入库、盘点和库存查询，帮助店主高效管理库存，降低人工成本，提升库存准确率。

### 1.2 产品目标
- 实现库存管理数字化，替代手工记账
- 通过扫码技术，提升入库、盘点效率50%以上
- 实时监控库存状态，避免缺货和积压
- 提供数据支持，辅助进货决策

### 1.3 目标用户
- **主要用户**：玩具店店主（1人）
- **次要用户**：店员（1人）

### 1.4 产品范围（MVP版本）
**包含功能**：
- 首页仪表盘（核心经营数据概览）
- 商品信息管理（含SKU自动生成）
- 二维码生成与打印
- 扫码入库（记录进货单价，加权平均成本）
- 扫码盘点（含定时盘点、审核流程）
- 实时库存查询（多端）
- 库存价值统计（成本价 + 零售价）
- 库存预警
- 进货建议
- 站内消息通知
- 数据导出（Excel）
- 操作日志
- 权限管理（角色可配置）
- 修改密码

**不包含功能（第二版迭代）**：
- 客如云销售对接（S5、S6）
- 销售统计分析（S9）
- 手机端扫码功能
- 供应商管理
- 销售利润分析

## 二、用户角色分析

### 2.1 店主
- **职责**：全面管理店铺，决策进货，查看经营数据
- **使用频率**：每天多次查看（手机端），每周到店1次
- **核心需求**：
  - 远程查看库存（手机端）
  - 查看库存价值、销售统计
  - 接收库存预警通知
  - 审核盘点差异
  - 管理商品和店员权限

### 2.2 店员
- **职责**：日常店铺运营，执行入库、盘点操作
- **使用频率**：每天使用（电脑端/平板端）
- **核心需求**：
  - 快速扫码入库
  - 快速扫码盘点
  - 查看库存数量（辅助销售）
  - 接收盘点任务提醒

## 三、功能需求

### 3.0 首页仪表盘模块

#### 3.0.1 首页仪表盘
**需求描述**：用户登录后，首页展示核心经营数据概览，快速掌握库存状况。

**功能要点**：
1. **统计卡片**
   - 库存总值（成本价）- 大数字卡片
   - 库存总值（零售价）- 大数字卡片
   - 商品种类数 - 小卡片
   - 库存预警数 - 小卡片（可点击跳转至预警列表）
   - 今日入库量 - 小卡片
   - 待审核盘点单 - 小卡片（可点击跳转至盘点审核）

2. **最近操作记录**
   - 显示最近5条操作记录
   - 包含操作时间、操作人、操作类型、操作详情

3. **数据刷新**
   - 页面加载时自动获取数据
   - 支持手动刷新
   - 数据每5分钟自动刷新

4. **多端适配**
   - 电脑端：网格布局，展示完整卡片
   - 手机端：纵向布局，展示核心卡片

**权限说明**：
- 店主：可查看所有数据
- 店员：仅查看商品种类数、今日入库量，不显示库存总值、预警数等敏感数据

### 3.1 商品管理模块

#### 3.1.1 商品信息维护
**需求描述**：店主能够录入、修改、查看商品信息。

**功能要点**：
1. **新增商品**
   - 系统自动生成SKU编码（格式：`KW-TOY-XXX`）
   - 必填字段：商品名称、分类、进货成本价、零售价
   - 选填字段：库存预警阈值、商品图片、商品描述
   - 保存后生成商品档案

2. **修改商品**
   - 可修改零售价、预警阈值、商品图片等
   - 进货成本价修改需记录日志（防止舞弊）
   - 已入库商品不能删除，只能"下架"

3. **商品列表**
   - 支持按名称/编码搜索
   - 支持按分类筛选
   - 支持按状态筛选（在售/下架）
   - 列表显示：SKU、名称、分类、当前库存、零售价

4. **商品详情**
   - 基本信息
   - 库存变动记录（入库、盘点、销售）
   - 加权平均成本（显示最新值）

**数据字段**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| SKU编码 | 字符串 | 是 | 系统自动生成，不可修改 |
| 商品名称 | 字符串 | 是 | 如"钢铁侠模型玩偶" |
| 商品分类 | 枚举 | 是 | 模型玩偶/模型汽车/模型机器人/其他 |
| 进货成本价 | 数值 | 是 | 初始值，后续按加权平均更新 |
| 零售价 | 数值 | 是 | 可修改 |
| 库存预警阈值 | 整数 | 否 | 默认0，表示不预警 |
| 商品图片 | 图片 | 否 | 支持上传 |
| 状态 | 枚举 | 是 | 在售/下架 |
| 创建时间 | 日期时间 | 是 | 系统自动记录 |
| 最后更新时间 | 日期时间 | 是 | 系统自动记录 |

#### 3.1.2 二维码生成与打印
**需求描述**：系统为每个商品生成二维码，支持批量打印。

**功能要点**：
1. **生成二维码**
   - 单个商品生成
   - 批量生成（选择多个商品）
   - 二维码内容：商品SKU编码
   - 格式：QR Code，容错等级M

2. **预览二维码**
   - 显示二维码图片
   - 显示商品信息（SKU、名称）
   - 支持调整打印尺寸（默认3cm x 3cm）

3. **打印二维码**
   - 选择打印机（热敏标签打印机）
   - 支持批量打印
   - 打印成功后标记"已打印"（避免重复打印）

4. **重新打印**
   - 支持已生成二维码的重新打印
   - 记录打印次数

**输出规格**：
- 尺寸：3cm x 3cm（可配置）
- 格式：PNG（预览） + 打印指令（ZPL/ESC/POS）
- 内容：仅包含SKU编码（如 `KW-TOY-001`）

### 3.2 入库管理模块

#### 3.2.1 扫码入库
**需求描述**：店员通过扫码枪扫描商品二维码，快速完成入库操作。

**功能要点**：
1. **创建入库单**
   - 系统自动生成入库单号（格式：`RK-YYYYMMDD-XXX`）
   - 自动记录操作人、操作时间

2. **扫码添加商品**
   - 扫码枪扫描商品二维码
   - 系统自动识别商品，添加到入库单明细
   - 支持手动输入数量（默认1）
   - 支持手动输入/修改进货单价（默认上次进货价）

3. **入库单明细管理**
   - 显示：商品SKU、名称、数量、单价、金额
   - 支持修改数量
   - 支持删除明细
   - 同一商品多次扫码，自动合并或提示累加

4. **提交入库**
   - 检查必填项（商品、数量、单价）
   - 提交后更新库存数量
   - 重新计算加权平均成本
   - 生成入库记录

5. **入库记录查询**
   - 按入库单号、日期、操作人筛选
   - 查看入库单详情
   - 支持导出

**业务流程**：
```
开始 → 新建入库单 → 扫码添加商品（输入数量、单价） → 重复扫码 → 提交入库 → 更新库存 → 结束
```

**数据字段（入库单头）**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 入库单号 | 字符串 | 系统自动生成 |
| 入库日期 | 日期 | 默认当天 |
| 操作人 | 字符串 | 当前登录用户 |
| 备注 | 字符串 | 选填 |
| 状态 | 枚举 | 待提交/已提交 |

**数据字段（入库单明细）**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 商品SKU | 字符串 | 关联商品档案 |
| 商品名称 | 字符串 | 自动带出 |
| 入库数量 | 整数 | 必须大于0 |
| 进货单价 | 数值 | 保留2位小数 |
| 金额 | 数值 | 数量 × 单价 |

**库存成本计算**：
- 采用移动加权平均法
- 公式：`新平均成本 = (原有库存金额 + 本次入库金额) / (原有库存数量 + 本次入库数量)`
- 每次入库后更新商品档案中的"加权平均成本"

### 3.3 盘点管理模块

#### 3.3.1 库存盘点
**需求描述**：店员通过扫码枪扫描商品，完成库存盘点，生成盘点差异单，店主审核后调整库存。

**功能要点**：
1. **创建盘点单**
   - 系统自动生成盘点单号（格式：`PD-YYYYMMDD-XXX`）
   - 选择盘点类型：临时盘点/定期盘点
   - 选择盘点范围：全盘/分类盘点/自定义盘点

2. **执行盘点**
   - 系统显示盘点清单（商品列表 + 系统库存）
   - 店员持扫码枪逐一扫描商品二维码
   - 每扫一个，系统将该商品的"已扫数量 +1"
   - 支持手动修改"已扫数量"
   - 支持"撤销上次扫码"

3. **完成盘点**
   - 点击"完成盘点"
   - 系统自动比对系统库存 vs 实际扫码数量
   - 生成盘点差异报告：
     - 盘盈（实际 > 系统）
     - 盘亏（实际 < 系统）
     - 相符（实际 = 系统）

4. **提交盘点单**
   - 盘点单状态变为"待审核"
   - 系统推送通知给店主

5. **审核盘点单（店主操作）**
   - 查看盘点差异详情
   - 选择审核结果：
     - **审核通过**：系统自动调整库存数量为实际数量
     - **审核驳回**：系统不调整库存，店员需重新盘点
     - **部分通过**：店主手动输入最终认可的库存数
   - 填写审核意见

6. **盘点记录查询**
   - 按盘点单号、日期、操作人、状态筛选
   - 查看盘点单详情和差异报告
   - 支持导出

**定时盘点功能**：
- 系统支持设置定期盘点计划（每周/每月）
- 在盘点日前1天推送提醒给店员
- 店员可提前开始盘点，或到期日当天盘点

**数据字段（盘点单头）**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 盘点单号 | 字符串 | 系统自动生成 |
| 盘点日期 | 日期 | 默认当天 |
| 盘点类型 | 枚举 | 临时盘点/定期盘点 |
| 盘点范围 | 枚举 | 全盘/分类盘点/自定义盘点 |
| 操作人 | 字符串 | 店员 |
| 审核人 | 字符串 | 店主 |
| 审核状态 | 枚举 | 待审核/审核通过/审核驳回 |
| 审核意见 | 字符串 | 选填 |

**数据字段（盘点单明细）**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 商品SKU | 字符串 | 关联商品档案 |
| 商品名称 | 字符串 | 自动带出 |
| 系统库存 | 整数 | 盘点开始时的库存 |
| 实际数量 | 整数 | 扫码数量 |
| 差异数量 | 整数 | 实际数量 - 系统库存 |
| 差异金额 | 数值 | 差异数量 × 加权平均成本 |

### 3.4 库存查询模块

#### 3.4.1 实时库存查询
**需求描述**：用户可以通过电脑端或手机端查看当前库存情况。

**功能要点**：
1. **库存列表**
   - 显示所有商品的库存信息：
     - 商品SKU、名称、分类
     - 当前库存数量
     - 加权平均成本
     - 库存金额（数量 × 加权平均成本）
     - 零售价总金额（数量 × 零售价）
     - 最后更新时间
   - 支持按名称/编码搜索
   - 支持按分类筛选
   - 支持按库存状态筛选（有库存/无库存/低库存）

2. **库存详情**
   - 基本信息
   - 入库记录（最近N条）
   - 盘点记录（最近N条）
   - 库存变动趋势图（如有销售数据）

3. **多端适配**
   - 电脑端：表格展示，支持导出Excel
   - 手机端：卡片式展示，支持搜索和筛选

**数据实时性**：
- 库存数据实时更新（入库、盘点后立即更新）
- 如无销售对接，则仅显示入库和盘点的变动

#### 3.4.2 库存价值统计
**需求描述**：店主可以查看当前库存的总价值，分别按进货成本价和零售价计算。

**功能要点**：
1. **价值统计报表**
   - 按进货成本计算：
     - 各商品库存金额（数量 × 加权平均成本）
     - 分类小计
     - 总计
   - 按零售价计算：
     - 各商品零售总额（数量 × 零售价）
     - 分类小计
     - 总计
   - 利润空间（零售总额 - 成本总额）

2. **筛选条件**
   - 按时间（截至今日、截至本月末、自定义）
   - 按分类

3. **导出功能**
   - 支持导出Excel
   - 包含明细和汇总数据

### 3.5 智能分析模块

#### 3.5.1 库存预警
**需求描述**：当商品库存低于设定的预警阈值时，系统自动提醒店主。

**功能要点**：
1. **设置预警阈值**
   - 在商品档案中设置"库存预警阈值"
   - 支持批量设置

2. **预警检查**
   - 系统每日检查库存
   - 如果当前库存 ≤ 预警阈值 → 触发预警

3. **预警通知**
   - 生成"库存预警列表"
   - 推送提醒给店主：
     - 手机端推送通知
     - 电脑端登录时弹窗提醒
   - 预警列表包含：
     - 商品名称、当前库存、预警阈值、建议进货量

4. **预警处理**
   - 店主可标记"已处理"（如已安排进货）
   - 支持"忽略预警"（如商品已停售）
   - 支持设置推送频率（每日1次/仅推送一次）

**预警逻辑**：
- 触发条件：当前库存 ≤ 预警阈值
- 预警级别：
  - 一般预警：库存 > 0
  - 紧急预警：库存 = 0

---

#### 3.5.2 进货建议
**需求描述**：系统根据销售速度和当前库存，自动计算建议进货量，辅助店主决策。

**功能要点**：
1. **计算建议进货量**
   - 销售速度 = 近30天销售数量 / 30（每日销售速度）
   - 可售天数 = 当前库存 / 销售速度
   - 建议进货量 = (进货周期 × 销售速度) - 当前库存
   - 如果建议进货量 < 0 → 不建议进货

2. **进货建议列表**
   - 商品名称、当前库存、近30天日均销量、可售天数、建议进货量、建议进货金额
   - 按可售天数排序（可售天数少的排前面）

3. **调整建议**
   - 店主可手动调整建议进货量
   - 生成"进货单草稿"（方便后续执行入库）

**说明**：
- 第一版可简化：仅根据"库存预警" + "销售速度"给出简单建议
- 如无销售数据，则仅根据"库存预警"给出建议

### 3.6 系统管理模块

#### 3.6.1 操作日志
**需求描述**：记录所有关键操作，店主可随时查看，便于追责和审计。

**功能要点**：
1. **记录操作日志**
   - 记录的操作类型：
     - 商品信息新增/修改
     - 入库操作（创建、提交）
     - 盘点操作（创建、提交、审核）
     - 库存调整（手动调整）
     - 权限变更
   - 记录内容：操作时间、操作人、操作类型、操作详情

2. **查看操作日志**
   - 按时间、操作人、操作类型筛选
   - 显示操作详情
   - 支持导出

**数据字段**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 操作时间 | 日期时间 | 精确到秒 |
| 操作人 | 字符串 | 用户姓名/账号 |
| 操作类型 | 枚举 | 商品新增/修改、入库、盘点、权限变更等 |
| 操作详情 | 字符串 | 如"入库单 RK-20260428-001，入库钢铁侠 x 10" |
| IP地址 | 字符串 | 选填，用于审计 |

---

#### 3.6.2 权限管理
**需求描述**：区分店主和店员的权限，支持按角色配置功能权限和数据权限。

**功能要点**：
1. **角色管理**
   - 系统预置角色：店主、店员
   - 支持自定义角色（未来扩展）
   - 每个角色可配置功能权限和数据权限

2. **功能权限**
   - 商品信息管理（无/查看/编辑）
   - 二维码生成与打印（无/查看/操作）
   - 入库管理（无/查看/操作/审核）
   - 盘点管理（无/查看/操作/审核）
   - 库存查询（无/查看）
   - 库存价值统计（无/查看）
   - 库存预警（无/查看/设置）
   - 进货建议（无/查看/调整）
   - 操作日志（无/查看）
   - 权限管理（无/管理）

3. **数据权限**
   - 查看范围（全部数据/仅自己创建的数据）
   - 操作范围（全部/仅自己）

4. **用户管理**
   - 店主可新增店员账号
   - 设置账号密码
   - 分配角色
   - 禁用/启用账号

#### 3.6.3 修改密码
**需求描述**：用户可以修改自己的登录密码。

**功能要点**：
1. **修改密码流程**
   - 输入旧密码（需验证正确性）
   - 输入新密码（需满足密码强度要求）
   - 确认新密码（两次输入需一致）
   - 点击"确认修改"

2. **密码强度要求**
   - 最少6个字符
   - 必须包含字母和数字
   - 新密码不能与旧密码相同

3. **安全策略**
   - 密码使用bcrypt加密存储
   - 修改成功后，当前Token不失效（本次登录有效期内）
   - 修改密码操作记录到操作日志

### 3.7 站内消息通知模块

#### 3.7.1 站内消息通知
**需求描述**：系统内重要事件通过站内消息通知用户，确保及时知晓库存预警、盘点审核等事项。

**功能要点**：
1. **通知类型**
   - 库存预警通知：商品库存低于预警阈值时，通知店主
   - 盘点审核通知：盘点单提交后，通知店主审核
   - 审核结果通知：盘点审核完成后，通知店员
   - 定时盘点提醒：定期盘点日前1天，提醒店员

2. **消息列表**
   - 显示所有消息（未读消息置顶）
   - 消息包含：标题、内容摘要、时间、是否已读
   - 导航栏显示未读消息数量（红点/数字）

3. **消息详情**
   - 点击消息查看完整内容
   - 提供"查看详情"按钮，跳转至相关页面
   - 消息标记为已读

4. **MVP版本限制**
   - 仅支持站内消息（登录后查看）
   - 不支持手机推送、微信通知
   - 消息保留30天，超期自动清理

**数据字段**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 通知ID | 整数 | 系统自动生成 |
| 接收用户ID | 整数 | 关联用户 |
| 通知类型 | 枚举 | 预警/审核/结果/提醒 |
| 通知标题 | 字符串 | 如"库存预警提醒" |
| 通知内容 | 字符串 | 详细描述 |
| 关联ID | 整数 | 关联的业务ID（如盘点单ID、商品ID） |
| 关联类型 | 枚举 | 如"盘点单"、"商品" |
| 是否已读 | 布尔 | 默认否 |
| 创建时间 | 日期时间 | 系统自动记录 |

#### 3.8.1 数据导出
**需求描述**：店主可以将系统中的各类数据导出为Excel文件。

## 四、非功能性需求

### 4.1 性能要求
- 扫码响应时间 < 1秒
- 库存查询加载时间 < 3秒（1000个商品以内）
- 入库单保存时间 < 2秒
- 支持同时100个商品扫码不卡顿

### 4.2 兼容性要求
- **电脑端**：Chrome浏览器（最新版）、Edge浏览器
- **手机端**：微信小程序 或 H5自适应页面（iOS Safari、Android Chrome）
- **扫码枪**：支持标准USB/蓝牙扫码枪（模拟键盘输入）

### 4.3 数据安全
- 操作日志记录所有关键操作
- 权限控制防止越权操作
- 数据每日自动备份
- 密码加密存储（bcrypt）
- HTTPS传输加密

### 4.4 可用性要求
- 系统可用性 ≥ 99%
- 支持离线操作（如扫码入库时网络中断，数据本地保存，网络恢复后自动同步）
- 操作失误可撤销（如入库单提交前可修改）

### 4.5 用户体验
- 界面简洁，操作流程不超过3步
- 扫码操作有声音/视觉反馈
- 重要操作有确认提示
- 错误提示清晰，指导用户如何修正
